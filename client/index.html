Collaborative map building and hangout/roleplay

About

Tilemap Town is a platform where you can build maps with your friends and roleplay and chat in the worlds you have created. It takes inspiration from BYOND "building games" that used to be popular long ago, as well as from MUCKs, Second Life, Pony Town, and other platforms.

You can explore a big world, and change tiles on it to build something nice and leave your mark on it. If you register an account (not required) you can make your own personal map or upload your own custom graphics for characters and map tiles. Want to help? Get in touch!

The web client will also work offline; you can work on maps privately and save them to your device.


Features

In Tilemap Town, you can wander around a top-down 2D pixel art world, and you can play as any character you like, as long as you can draw or find a 32x32 sprite of them! There is support for multiple directions and simple animation, and there's a guide on the tutorials page that explains how custom player graphics works; for the most part you just grab a template and fill it in.

Tilemap Town's world consists of maps, which are 2D grids of tiles that can be modified by players, letting them collaboratively make a world together. If you register then you can even create your own maps. You can restrict building to yourself only, limit entry to specific users, and have a private space if you would like one.

Map features:
  • You can use your own graphics for the tiles you draw onto maps!
  • Small amount of space to upload your own graphics and music to
  • Tons of premade tiles available to build with
  • Autotiling and animation support
  • Can click and edit the properties of tiles directly on the map
  • Place objects in the world to act as props
  • Maps can be linked to each other, so that you can pass over the edge of one and end up on another, to make a big connected world
Chat features:
  • BBCode for chat messages, with support for multiple lines of text
  • Helpful commands like dice rolls and teleport requests
  • Multiple commands to help with roleplaying: /me /ooc /spoof
  • Tries to gracefully handle momentarily losing connection to the server; sent messages and commands should reach their destination once and only once
Other features:
  • An inventory system, allowing you to hold onto objects and keep notes for yourself
  • A simple mail feature (and private messages sent to offline users will be held until they're online)
  • A profile feature where you can write about yourself
  • Save multiple characters and swap between them easily with commands
Some advanced features:
  • Scripting with the Luau language (which requires a permission given by the admin) - see the scripting documentation
  • Documented protocol, with an open source client and server
  • An elaborate permission system, including the ability to grant permissions to someone temporarily
  • Features intended to make bots easier, where you can remotely interact with other maps (if granted permission to)
  • There's support to let bots and scripts run their own little minigames directly on the map, with their own screens you can click on

Community

There is an official server that's been running since 2017, and it's not especially lively (yet!) but there is quite a lot to explore. If you do want to meet people your best chance is on Tuesday evenings (Eastern Time), which is "Tilemap Town Tuesday." You're welcome to come visit and check it out, and even leave your own mark on the world by building something nice if you like. The official server is the default one on the web client!

The community is pro-LGBT+ and is meant to be a safe space to hang out. Be kind to the other people you meet here. Note that it's not really intended to be a space for kids.

There's also a small Discord server for Tilemap Town and other projects I work on, but like Tilemap Town you should keep in mind that it is meant to be a SFW environment for adults. You can chat with other people interested in Tilemap Town and plan out events/hangouts. You can also get updates about what's been going on through the Tilemap Town blog, which has RSS and Atom feeds available.

If you prefer forums, you can post about Tilemap Town here. I can give Tilemap Town its own forum if there's demand and people would actually use it.

Roleplay setting

The official server is intended to be a SFW furry roleplay space, like MUCKs usually are, but you're welcome to just hang out and chat if that's more your thing! I think it would be interesting to really embrace the nature of the platform, and treat it in-character as a digital world where it's especially easy to shape the land to your wishes and build things.

The world is sort of a crossroads where characters from lots of other worlds meet up, sometimes because they had to leave home for whatever reason and needed a place to go, like Traverse Town. Fantasy elements and magic are encouraged too, and the tone is supposed to be lighthearted. Explore the world, introduce people to your character, and add your own (literal) worldbuilding.

Privacy notice

The official server is configured to log when users connect to the server, modify a map, or upload files, for moderation and review purposes. The logs contain the user's IP address and what website they connected from. Chat messages (both private messages and messages sent to maps) are not logged, and are discarded once they reach their destination. Tilemap Town does not use end-to-end encryption, so you should not send sensitive information on this platform.

Private messages sent to offline users are temporarily kept in memory (not the database) until the recipient logs in, at which point they will be discarded. However, if the server is restarted while there are still private messages waiting to be sent, they will be converted to mail. Mail is stored in the database and is not currently encrypted. /userflags add no_offline_pm_mail will opt out of this feature for both messages you send and messages sent to you, causing messages to simply be discarded in this case.

Open platform

Tilemap Town is an open source project, and you can find the web client, server, and documentation on GitHub. Anyone can host their own server or make their own client, or customize the official server or client to their liking. It could even be a good head start if you're trying to make your own online game.

The server is written in Python, so it's a lot easier to extend and change than the old MUCK codebases from the 90s. The database is SQLite so there's no setup, and you can use preexisting tools to browse and interface with the database.

Tilemap Town is designed with making your own clients and bots in mind, and that's encouraged! The protocol is built on top of WebSockets and JSON, which most languages should be able to use without too much trouble, and there's advice in the documentation about what a useful subset of the protocol would be. It's designed with the idea of progressive enhancement and is meant to work well with a minimal client, while allowing more advanced clients to have access to fancy features.

In a way, Tilemap Town is sort of like IRC, in that it's decentralized and easy to connect to and interface with, but its design incorporates some modern features and leaves room for things like metadata and longer messages without needing that stuff to be in the form of extensions.

There's also a simple HTTP API you can hit to find out some basic information about who's currently online. The official server has its API exposed here.
(You can opt out of showing up on this with /userflags add hide_api)

Some projects that interface with Tilemap Town

  • Desktop client, made with Qt 6 in C++; has very basic features where you can explore and chat, but not build yet. Part of the goal of this client is to have a reusable C++ core for handling the protocol across clients for different platforms, and the protocol code is kept separate from the UI code where feasible.
  • 3DS client, using the same core as the desktop app with similar features, also in C++. Currently crashy and map rendering is a bit buggy.
  • MU* gateway, which allows you to connect through a MU* client. It allows you to use commands to view a "graphical" version of the map made out of text, and is really mostly a proof of concept.
  • Example bot that implements a very small portion of the protocol, enough to receive and respond to messages.
